## 4. Server Connection

### For communication with the central server, the traffic light uses an MQTT v5 client. Upon system initialization, the Raspberry Pi Pico 2W board establishes a TCP connection to the server on port 5050. Once the connection is established, a CONNECT packet is sent. If the authentication and parameters are correct, the server responds with a CONNACK packet, confirming the connection setup.

### After establishing the connection, the traffic light subscribes to two types of MQTT topics:
- General topic for traffic lights: `test/masini`
- Individual topic dedicated to each traffic light: `test/traffic_light_name` (where `traffic_light_name` is the unique identifier of the traffic light)

### Effective communication is carried out using PUBLISH packets, using QoS 1 for increased reliability. The traffic light constantly listens to these topics and, upon receiving relevant messages, executes the corresponding actions.

---

### Types of messages handled by the traffic light:

1. **Changing the Traffic Light State**
    - The server can send a message containing the current state of the traffic light, encoded for each direction, for example:
      ```
      r g
      ```
      where `r` represents red and `g` represents green for each traffic direction.
    - The traffic light will turn on the corresponding LEDs for each direction based on the received message.

2. **Operating Mode Change Commands**
    - The server can send messages to change the operating mode:
      - `m` for manual mode
      - `s` for sequential mode
      - `i` for intelligent/adaptive mode
    - The traffic light will change the operating mode immediately according to the command.

3. **Configuring Time Parameters**
    - For sequential mode, the server can send the operating times for each color, for example:
      ```
      s d1 r 7 y 2 g 5
      ```
      (sequential mode, direction 1, red 7 seconds, yellow 2 seconds, green 5 seconds)
    - Parameters can be dynamically customized to adapt to various traffic situations.

4. **Monitoring Messages**
    - The traffic light can transmit information back to the server about its current state (e.g., which color is active in each direction) or about detected events (such as the presence of cars at the IR sensors).
    - These messages allow for centralized monitoring and data archiving for later analysis.

---

### Summary

Through integration with the MQTT server, the traffic light can be controlled and monitored remotely in real-time. This mechanism allows for:
- Rapid changes to the operating mode and parameters,
- Coordination with other traffic lights or traffic participants (via common topics),
- Centralization of data for further analysis and optimization.
