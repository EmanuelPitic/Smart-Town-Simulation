## 2. Application Design

### To enable remote control and monitoring of the traffic light, the application uses an MQTT client to communicate with the central server. The MQTT client is managed through the MQTTClient class, and the entire operating logic of the traffic light is encapsulated in the TrafficController class, which ensures both local control (via Bluetooth and sensors) and remote control (via MQTT).

### The class diagram below highlights how the main software components of the traffic light application are organized:

![Traffic Light Class Diagram](/Documentation/Traffic%20Light/Images/traffic_light_class_diagram.png)

---

### Description of Main Classes:

- **MQTTClient** Manages the connection with the MQTT broker, sending and receiving messages from the server, as well as subscribing to relevant topics. This allows the synchronization of the traffic light state with the central system.

- **UARTManager** Ensures communication via UART with the HC-05 Bluetooth module, allowing local commands to be sent and received. It is used for manual control and for testing the system in the absence of a Wi-Fi connection.

- **WiFiManager** Manages the connection to the Wi-Fi network and enables communication with the MQTT server. If the connection cannot be established, the application continues to function locally (Bluetooth).

- **TrafficLight** Models a physical traffic light, managing the LED states (red, yellow, green) for a specific direction. It allows individual control of the traffic light for each traffic flow.

- **Sensor** Represents an infrared sensor used to detect the presence of vehicles in front of the traffic light. Information from the sensors is used for the intelligent operating mode.

- **TrafficController** The main class that implements the traffic light's operating logic, including all operating modes (manual, sequential, intelligent). It receives commands from MQTT, Bluetooth, or sensors and updates the traffic light state accordingly.

- **App** The "manager" type classes are instantiated and coordinated by this class, which manages asynchronous tasks for control, communication, and system monitoring.

---

The use of these classes allows for application modularity, facilitating both local testing and easy integration with the network infrastructure (via MQTT) for centralized monitoring and control.
