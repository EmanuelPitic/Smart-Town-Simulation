## 1. Introduction
### The aim of this project is to develop an autonomous car controlled by a Raspberry Pi Pico W board (revision 2, labeled "Pico 2W"). It is responsible for managing the sensors, motors, and communication with an external server via the MQTT (Message Queuing Telemetry Transport) protocol. The car is designed to autonomously follow a route defined by a black line (when in automatic mode), detect obstacles or traffic lights, and switch between two operating modes: automatic and manual.

### The Raspberry Pi Pico 2W is an efficient and compact solution based on the RP2040 microcontroller, offering Wi-Fi connectivity, low power consumption, and the ability to interact directly with sensors and actuators essential for the car's autonomous operation. Choosing this board ensures an optimal balance between performance, cost, and flexibility.

### In automatic mode, the car uses line sensors to follow the track. In manual mode, the user can send real-time control commands through an MQTT interface, for example, from a web client. Ultrasonic sensors detect obstacles or traffic lights and signal the car to stop.

### Communication via MQTT allows for a flexible and scalable architecture, suitable for IoT applications and small-scale autonomous vehicles. Data such as the detected distance, current operating mode state, and track events can be transmitted to the server for monitoring or further analysis.

### By combining detection functionalities, wireless communication, and automatic control, the project highlights a practical application of a cyber-physical system, contributing to the understanding of hardware and software integration in a modern embedded system.
