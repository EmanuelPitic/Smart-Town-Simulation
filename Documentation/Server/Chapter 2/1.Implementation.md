## 1. Implementation 

The application was developed using the Python programming language based on Object-Oriented Programming (OOP) principles. Standard libraries such as `socket`, `struct`, and `sqlite3` were used for connection management, packet processing, and data storage. The modular structure ensures a clear separation of functionalities, making each part of the code easier to manage and modify.

### Module Description 

1. **Main Server (server.py):** - Manages client connections using sockets. 
- Decodes MQTT packets using the `MQTTDecoder`. 
- Manages **PUBLISH**, **SUBSCRIBE**, and other MQTT packets according to MQTT v5 standards. 
- Allows for a safe server shutdown using a `shutdown_event`. 
- The MQTT server is started from the graphical interface and listens on port 5050.

![Server Class Diagram](/Documentation/Server/Images/server_class_diagram.jpg)

2. **MQTT Client (MQTTClient.py):** - Simulates an MQTT client that connects to the broker on port 5050.
- Interprets received messages and acts according to their content (e.g., commands for the autonomous car).
- Is started separately and runs on port 5000.

![Client Class Diagram](/Documentation/Server/Images/client_class_diagram.jpg)

3. **Web Clients Controller (controller.py):**
- A web application that allows the control and monitoring of subscribers connected to the MQTT server, specifically key devices like the autonomous car and the traffic light system.
- Communicates with both the graphical interface and the MQTT server to manage message flow and subscriber status.

4. **Web Server Controller (mqtt_webapp.py):**
- The web interface offers complete control over the MQTT server, allowing it to be started and stopped on port 5050.
- Runs on port 8082 and communicates with the GUI and the MQTT server to manage message flow.
- Facilitates real-time management and oversight of the entire MQTT network, providing a clear view and rapid access to system status.
- Through this interface, useful system information can be monitored, such as:
    + Topic history
    + Last transmitted messages
    + Currently connected clients 
    + Clients subscribed to various topics

![MQTT Dashboard](/Documentation/Server/Images/mqtt_dashboard.jpg)

5. **Packet Decoding (decoder.py):** - Decodes packet types like CONNECT, PUBLISH, SUBSCRIBE, etc. 
- Extracts information regarding topics, QoS, and payload. 

6. **Message Dispatcher (message_dispatcher.py):** - Ensures message delivery based on subscriptions. 
- Manages QoS for packets requiring acknowledgments (ACKs). 

7. **Database (sqlServer.py):** - Stores clients, topics, subscriptions, and messages. 
- Uses **SQLite** as the data storage platform. 

8. **Graphical User Interface (gui.py):** - Provides sections such as topic history, the last 10 messages, active connection visualization, subscriptions, and QoS messages. 
- Allows for the safe starting and stopping of the server. 

9. **MQTT Packet Creation (packet_creator.py):** - Provides functions for generating packets such as CONNACK, PUBACK, PINGRESP, SUBACK, UNSUBACK, DISCONNECT, etc. 

### Database Structure 
- **clients** table: Stores client information (ID, connection status, disconnection time). 
- **subscriptions** table: Keeps track of client topic subscriptions. 
- **messages** table: Stores published messages, including QoS level and publication time. 
- **topics** table: Defines topics and retained messages.
