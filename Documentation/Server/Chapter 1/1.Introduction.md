## 1. Introduction

In the era of interconnectivity and automation, efficient and reliable communication protocols play a crucial role in implementing intelligent systems. MQTT (Message Queuing Telemetry Transport) is one of the most popular protocols used in IoT (Internet of Things) applications, appreciated for its simplicity, efficiency, and low resource consumption.

This project aims to implement a fully functional MQTT v5 system, simulating a smart network of devices that communicate with each other in an asynchronous and efficient manner. At the heart of this network lies an MQTT v5 broker, developed and running on a Raspberry Pi device, acting as an intermediary for message exchange between various entities.



The system architecture includes the following major components:

- **An MQTT Server (Broker):** Implemented in Python, running on a Raspberry Pi Zero 2 W. It manages client connections, topic subscriptions, and message distribution.

- **An Autonomous Car:** Controlled by a Raspberry Pi Pico W, functioning as an MQTT client in the network. It can be operated in two modes: **Manual**, where the server transmits direct commands (right, left, backward, forward) for direction control, and **Autonomous**, where the server activates an automatic line-following mode without further intervention. Switching between the two modes is done via MQTT messages, and the car's behavior adapts based on the commands received from the broker.

- **A Traffic Light Intersection Control System:** Also functioning as an MQTT client, interacting with the rest of the network. The state of the traffic lights is dictated by messages received through the broker, and current signals can be transmitted to other entities in the network (e.g., the car) for synchronization and safety.
